# System Requirements

## Overview
The Knowledge Rotation System is designed to help organizations identify
critical knowledge dependencies and reduce operational risks caused by
employee turnover or temporary absences.

## User Roles
- Admin
  - Manage areas and employees
  - Assign knowledge to employees
  - View risk indicators
- Manager
  - View area knowledge coverage
  - Identify key personnel
- Viewer
  - Read-only access to dashboards

## Functional Requirements
- Create, update, and deactivate employees
- Create and manage areas / departments
- Register knowledge items
- Assign knowledge to employees with a proficiency level
- Mark employees as active or leaving
- Calculate risk level per area
- Display risk indicators in a dashboard

## Non-Functional Requirements
- Role-based access control
- Secure authentication
- Scalable architecture
- Clear separation between frontend and backend
- RESTful API design

## Constraints
- Web-based system
- Relational database
- English documentation and codebase
